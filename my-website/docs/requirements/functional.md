# Функциональные требования

> Все ФТ написаны для MVP-версии

## FR-001 — Создание и публикация турнира

| Раздел | Описание |
|---|---|
| ID | UC-1 |
| Акторы | Организатор (авторизованный), Система |
| Предусловия | Пользователь зарегистрирован, авторизован и имеет роль «Организатор» |

### Постусловия

**Успех:** турнир сохранён в БД, присвоен уникальный ID, статус = «Опубликован», турнир отображается в каталоге.

**Неудача:** турнир не создан, пользователю отображается сообщение с указанием ошибки.

### Основной поток

1. Организатор открывает раздел «Создать турнир»
2. Система отображает форму создания
3. Организатор заполняет: название (3–100 символов), дисциплину, формат (LAN/Online), дату (≥ текущей), лимит команд (≥ 2)
4. Организатор нажимает «Создать»
5. Система валидирует данные
6. Система сохраняет турнир со статусом «Черновик»
7. Организатор нажимает «Опубликовать»
8. Система меняет статус на «Опубликован» и добавляет турнир в каталог

### Бизнес-правила

- **BR-01:** В MVP поддерживается только формат Single Elimination
- **BR-02:** Турнир может создать только пользователь с ролью «Организатор»
- **BR-03:** Все обязательные поля должны быть заполнены

## FR-002 — Поиск и фильтрация турниров

| Раздел | Описание |
|---|---|
| ID | UC-2 |
| Акторы | Игрок (гость или авторизованный), Система |
| Предусловия | В системе есть минимум один опубликованный турнир |

### Основной поток

1. Игрок открывает раздел «Каталог турниров»
2. Система отображает список турниров со статусом «Опубликован» и панель фильтров
3. Игрок выбирает фильтры: дисциплина, регион, формат (LAN/Online)
4. (Опционально) Игрок вводит текст в строку поиска
5. Система отображает турниры, соответствующие всем выбранным параметрам

### Бизнес-правила

- **BR-01:** В каталоге отображаются только турниры со статусом «Опубликован»
- **BR-02:** При выборе нескольких фильтров отображаются только турниры, соответствующие всем параметрам одновременно

## FR-003 — Регистрация команды на турнир

| Раздел | Описание |
|---|---|
| ID | UC-3 |
| Акторы | Игрок (капитан команды, авторизованный), Система |
| Предусловия | Турнир опубликован, регистрация открыта, игрок авторизован |

### Основной поток

1. Игрок открывает страницу турнира
2. Нажимает «Зарегистрировать команду»
3. Заполняет форму: название команды, состав (никнеймы), контакт капитана (email)
4. Система валидирует данные и проверяет лимит команд
5. Система сохраняет команду и отображает «Регистрация успешно завершена»

### Бизнес-правила

- **BR-01:** Одна команда может быть зарегистрирована на турнир только один раз
- **BR-02:** Регистрация невозможна после достижения лимита команд
- **BR-03:** Минимальный состав команды для командных дисциплин = 5 игроков

## FR-004 — Ввод результатов матчей

| Раздел | Описание |
|---|---|
| ID | UC-4 |
| Акторы | Организатор (авторизованный), Система, Игрок (просмотр) |
| Предусловия | Турнирная сетка сформирована, турнир в статусе «В процессе» |

### Основной поток

1. Организатор открывает страницу сетки турнира
2. Выбирает завершённый матч и нажимает «Ввести результат»
3. Вводит счёт в формате «X:Y» (целые числа ≥ 0)
4. Система сохраняет результат, определяет победителя и продвигает его в следующий раунд
5. После финального матча — турнир переходит в статус «Завершён»

### Бизнес-правила

- **BR-01:** Результат может вводить только организатор турнира
- **BR-02:** После завершения турнира изменение результатов невозможно
- **BR-03:** Победитель — команда с большим количеством очков

## FR-005 — Генерация турнирной сетки

| Раздел | Описание |
|---|---|
| ID | UC-5 |
| Акторы | Организатор (авторизованный), Система |
| Предусловия | Турнир опубликован, регистрация закрыта, зарегистрированы минимум 2 команды |

### Основной поток

1. Организатор открывает страницу управления турниром
2. Нажимает «Сгенерировать сетку»
3. Система формирует сетку формата Single Elimination
4. Команды распределяются по парам в порядке регистрации
5. Сетка отображается на странице турнира

### Бизнес-правила

- **BR-01:** В MVP поддерживается только Single Elimination
- **BR-02:** Сетка может быть сгенерирована только после закрытия регистрации
- **BR-03:** После начала матчей перегенерация невозможна
