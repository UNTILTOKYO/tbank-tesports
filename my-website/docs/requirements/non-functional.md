# Нефункциональные требования

> НФТ реализованы для MVP

## Таблица требований

| ID | Категория | Требование | Приоритет |
|---|---|---|---|
| NFR-001 | Безопасность | Передача данных по HTTPS (TLS 1.2+). Пароли хранятся в виде хеша (bcrypt, cost ≥ 12). Доступ к функциям организатора только для авторизованных пользователей с соответствующей ролью | MUST |
| NFR-002 | Производительность | Время загрузки страниц «Каталог», «Страница турнира», «Сетка» ≤ 2 секунд (p95) при нагрузке до 200 одновременных пользователей | SHOULD |
| NFR-003 | Производительность (нагрузка) | Система обрабатывает до 100 HTTP-запросов в секунду при p95 времени ответа ≤ 500 мс и доле ошибок 5xx ≤ 1% | MUST |
| NFR-004 | Надёжность | Доступность системы ≥ 99% в месяц. Резервное копирование БД ежедневно, срок хранения ≥ 14 дней | MUST |
| NFR-005 | Масштабируемость | Архитектура поддерживает до 1 000 одновременных пользователей и горизонтальное масштабирование (≥ 2 экземпляра без изменения бизнес-логики) | SHOULD |
| NFR-006 | Наблюдаемость | Логируются: создание турнира, регистрация команды, генерация сетки, ввод результатов, ошибки 5xx. Логи хранятся ≥ 30 дней | SHOULD |
| NFR-007 | Удобство использования | Адаптивный интерфейс (320px–2560px). Основной сценарий регистрации команды выполняется не более чем за 5 шагов | SHOULD |
| NFR-008 | Совместимость | Веб-приложение корректно работает в браузерах на базе Chromium (последние 2 версии) | MUST |

## Почему 100 RPS?

При целевом DAU = 200 пиковая одновременная активность оценивается как ~80 пользователей. В пике суммарная нагрузка может достигать ~90 RPS. С учётом запаса устойчивости задаётся требование **100 RPS**.

## Непокрытые категории НФТ

### Capacity
Не включена в MVP: ожидаемый DAU ≈ 200, ≤ 20 турниров в месяц, нет хранения медиа и тяжёлых логов. Будет актуализирована в версии 2.0+.

### Regulatory
Не включена в MVP: нет платёжной системы и обработки банковских данных. При добавлении монетизации потребуется соответствие 152-ФЗ и 54-ФЗ. Планируется в версии 3.0.

### Maintainability
Не включена в MVP: проект реализуется небольшой командой, инфраструктура в упрощённой конфигурации. Планируется в версии 2.0+: покрытие автотестами ≥ 70–80%, CI/CD, документирование API (OpenAPI/Swagger).
